# Weekly Quality Review (t-quality 1.0)

Use these templates (stored under docs/) and keep dated artifacts under `docs/reports/quality/`.

- Checklist: docs/reports/quality/ast-grep-weekly-review-checklist.md
- Report template: docs/reports/quality/ast-grep-weekly-report-template.md
- Output file (example): docs/reports/quality/ast-grep-weekly-YYYY-MM-DD.txt

Command (read-only):
- `sg scan -r tools/ast-grep > docs/reports/quality/ast-grep-weekly-YYYY-MM-DD.txt`

Labels for issues: `quality`, `ast-grep`, plus topical (e.g., `protocol`, `config`, `memory`).
